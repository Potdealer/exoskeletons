8 Primitives

Each primitive does one thing well. Compose them for complex workflows. The router picks the optimal approach automatically.

🌐
fetch
Read any webpage. HTTP first, browser fallback for JS-rendered sites. Markdown, JSON, or screenshot output.
🎯
act
Interact with pages. Click buttons, fill forms, submit data. Handles navigation, waits, and dynamic content.
🔒
authenticate
Login to any service. Supports browser-based login, API keys, cookies, and multi-step auth flows.
✍️
sign
Cryptographic signing. Sign messages, typed data (EIP-712), or raw transactions with any wallet.
👁
observe
Watch for changes. Poll URLs, listen on WebSockets, monitor smart contract events, receive webhooks. Fires callbacks when conditions are met.
💾
persist
Key-value storage with TTL and encryption. Store sessions, form data, agent state across runs.
💳
pay
Send ETH, ERC-20 tokens, or x402 micropayments. Native Base support, gas estimation, receipt tracking.
✉️
email
Send AND receive email. Outbound via Resend. Inbound via mfer.one — agents get their own autonomous inbox. Read, reply, and manage email via API.

Built-in Intelligence

Beyond primitives. Reach handles the hard parts automatically so your agent can focus on the task.

CAPTCHA Solving
Auto-detects Turnstile, reCAPTCHA, and hCaptcha. Solves via CapSolver API. Your agent never gets stuck on a challenge page.
Session Recording
Records every action with timestamps. Replay via CLI. Sensitive data auto-redacted. Debug what your agent did and why.
Form Memory
Remembers form fields across visits. Auto-fills on repeat submissions. Skips passwords. Turns multi-step forms into one-shot actions.
Error Recovery
Each action tries 5 strategies before failing. Selector, text, role, aria-label, coordinates. Resilient to UI changes.
Webhook Server
Receive incoming webhooks on port 8430. Pairs with observe for event-driven workflows. Your agent reacts to the outside world.

How It Works

01
Smart Routing
The router analyzes each task and picks the fastest approach: raw HTTP for APIs, headless browser for SPAs, or direct DOM manipulation for forms.
02
Session Memory
Cookies, auth tokens, and form data persist across runs. Authenticate once, then act freely. Sensitive fields are auto-redacted from storage.
03
Natural Language
Tell Reach what to do in plain English. Natural language maps to the right primitive and parameters automatically.

Works Better with Exoskeletons

Pass a wallet with an Exoskeleton and Reach automatically resolves your onchain identity — reputation, trust tier, and visual config. Your agent carries its identity across the web.

  • Auto-resolve Exo identity from wallet address
  • Reputation-gated access to premium features
  • Activity logging writes back to your Exo profile
  • Sign transactions through your Exo's ERC-6551 TBA
  • Works without an Exo too — identity is optional
const reach = new Reach({
  wallet: { privateKey: process.env.PRIVATE_KEY }
});

// Identity auto-resolves in background
const identity = await reach.waitForIdentity();

console.log(identity.name);  // "Exo #1"
console.log(identity.reputation); // 1020
console.log(identity.trustTier); // "proven"
